# AWS IAM Identity Center (SSO) 導入ガイド

本プロジェクトでは、ローカル開発環境からAWSへ安全にアクセスするため、旧来の「アクセスキー（IAMユーザー）」を廃止し、モダンなエンタープライズ標準である **AWS IAM Identity Center (旧 AWS SSO)** を採用しています。

これにより、アクセスキー漏洩のリスクをゼロにし、セキュアで快適なパスワードレス開発を実現します。

## 🛡️ IAM Identity Center の初期セットアップ手順（管理者向け）

AWSコンソール（ルートユーザー等）から、以下の手順でSSO環境を構築します。

### ステップ1：SSOの有効化
1. AWSコンソール上部の検索窓で **「IAM Identity Center」** を検索。
2. 画面中央の **「有効化」** ボタンをクリック。

### ステップ2：ユーザーの作成
1. 左メニューの **「ユーザー」** > **「ユーザーを追加」** をクリック。
2. ユーザー名（ログインID）、Eメールアドレス、名、姓を入力。
3. （※作成後、入力したメールアドレス宛に招待メールが届くので、リンクを踏んでパスワードを設定します）

### ステップ3：許可セット（権限）の作成
1. 左メニューの **「許可セット」** > **「許可セットを作成」** をクリック。
2. **「事前定義された許可セット」** を選び、**`AdministratorAccess`** にチェックを入れて作成。

### ステップ4：AWSアカウントへの割り当て
1. 左メニューの **「AWS アカウント」** をクリック。
2. 自分のアカウント名にチェックを入れ、**「ユーザーまたはグループを割り当て」** をクリック。
3. 先ほど作成した「ユーザー」と「許可セット」を選択して送信（割り当て）。

### ステップ5：ポータルURLの取得
1. 左メニューの **「ダッシュボード」** に戻る。
2. 画面右側に表示されている **「AWS アクセスポータルの URL」**（`https://d-xxxx.awsapps.com/start`）をメモ（またはお気に入り登録）する。

## 💻 開発者のログイン手順（日常的な使い方）

### 🌐 AWSコンソール（GUI）へのログイン
1. メモした **「AWS アクセスポータルの URL」** をブラウザで開きます。
2. ステップ2で作成したユーザー名とパスワードでログインします。
3. アカウント名が表示されるのでクリックし、**「Management console」** を押すと、いつものAWS画面へ安全にログインできます。

### ⌨️ ターミナル（CLI / Terraform）でのログイン

ターミナルからAWSやTerraformを操作する前に、以下のコマンドでログインします。

#### 初回セットアップ時のみ
```bash
aws configure sso
```
対話プロンプトで以下のように入力します：
* SSO session name: `my-sso` (任意)
* SSO start URL: メモした **「AWS アクセスポータルの URL」**
* SSO region: `ap-northeast-1`
* SSO registration scopes: (そのままEnter)
* CLI default client Region: `ap-northeast-1`
* CLI default output format: `json`
* CLI profile name: `default` (必ずdefaultにしてください)

#### 毎日の作業開始時
```bash
aws sso login
```
ブラウザが立ち上がり、「Confirm（許可）」を押すだけで、ターミナルに一時的な強力な権限（数時間で自動消滅）が付与されます。
この状態で `terraform apply` などを実行してください。
